SECTION 1 - THEORY

1. What is FastAPI and why is it popular for building backend applications?

Student Attempt:
FastAPI is a Python framework for creating APIs. It is popular because it is simple and fast, and it can create documentation automatically.

Comment:
This is mostly correct.

2. Explain what an HTTP request lifecycle is inside a FastAPI application.

Student Attempt:
An HTTP request lifecycle is when the browser sends a request, FastAPI receives it, runs the correct function, and returns a response.

Comment:
Mostly correct, but it misses middleware, validation, and response processing.

3. What is a middleware in FastAPI and when would you use it?

Student Attempt:
Middleware is a function that runs only after the route is finished. It is used mostly for printing logs.

Comment:
Partly incorrect. Middleware can run before the route and after the route. It can be used for logging, CORS, authentication checks, headers, and timing.

4. What is the purpose of request body parsing in an API?

Student Attempt:
Request body parsing is used to read JSON data sent by the client and convert it into Python data.

Comment:
Correct, but a stronger answer would also mention validation.

SECTION 2 - MULTIPLE CHOICE

Student Attempt:

Comment:
Correct.

2. What decorator is used to define a POST route in FastAPI?

Student Attempt:

Correct.

Student Attempt:

SECTION 3 - CODING

1. Create a simple FastAPI app with one route /hello that returns { "message": "Hello, Level 4!" }.

Student Attempt:

from fastapi import FastAPI

app = FastAPI()

@app.get("/hello")
def hello():
    return {"message": "Hello, Level 4!"}

Comment:
Correct.

2. Write a POST endpoint /user that accepts JSON data with "name" and "age" and returns the same data.

Student Attempt:

from fastapi import FastAPI

app = FastAPI()

@app.post("/user")
def create_user(name: str, age: int):
    return {"name": name, "age": age}

Comment:
Partly incorrect. This accepts query parameters, not a JSON body. A better answer should use a Pydantic model.

Correct version:

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class User(BaseModel):
    name: str
    age: int

@app.post("/user")
def create_user(user: User):
    return user

3. Write a simple middleware that prints "Request received" before handling any route.

Student Attempt:

from fastapi import FastAPI, Request

app = FastAPI()

@app.middleware("http")
async def middleware(request: Request, call_next):
    print("Request received")
    response = await call_next(request)
    return response

Comment:
Correct.

SECTION 4 - ALGORITHMS

1. Bubble Sort algorithm.

Student Attempt:
Bubble Sort checks each pair of neighboring values and swaps them if they are in the wrong order. It repeats this until the array is sorted.

Pseudocode:

for i from 0 to length(array):
    for j from 0 to length(array):
        if array[j] > array[j + 1]:
            swap array[j] and array[j + 1]

Comment:
Explanation is correct, but pseudocode has an index error because j + 1 can go outside the list. The inner loop should stop earlier.

Correct pseudocode:

for i from 0 to length(array) - 1:
    for j from 0 to length(array) - i - 2:
        if array[j] > array[j + 1]:
            swap array[j] and array[j + 1]

2. Insertion Sort algorithm.

Student Attempt:
Insertion Sort takes each element and places it into the correct position in the sorted part of the array.

Pseudocode:

for i from 1 to length(array) - 1:
    key = array[i]
    j = i - 1

    while j >= 0 and array[j] > key:
        array[j + 1] = array[j]
        j = j - 1

    array[j + 1] = key

Comment:
Correct.

SECTION 5 - DEBUGGING

Original code:

from fastapi import FastAPI
app = FastAPI
@app.get("/ping")
def ping():
return "pong"
@app.post("/user")
def create_user(name: str, age: int):
return {name: name, age: age}

Student Attempt:

from fastapi import FastAPI

app = FastAPI()

@app.get("/ping")
def ping():
    return "pong"

@app.post("/user")
def create_user(name: str, age: int):
    return {"name": name, "age": age}

Comment:
Partly correct. It fixes FastAPI() and indentation. However, the /user endpoint still uses query parameters instead of a request body. A stronger answer should use Pydantic.

Fully corrected version:

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class User(BaseModel):
    name: str
    age: int

@app.get("/ping")
def ping():
    return "pong"

@app.post("/user")
def create_user(user: User):
    return {"name": user.name, "age": user.age}


SUMMARY OF INTENTIONAL PRACTICE MISTAKES

1. Middleware theory says it runs only after the route.
2. Multiple choice Section 2, Question 3 is wrong.
3. POST /user coding answer uses query parameters instead of JSON body.
4. Bubble Sort pseudocode has an index range problem.
5. Debugging answer fixes some mistakes but does not use a Pydantic model.

This file is for studying common errors and understanding how to correct them.
